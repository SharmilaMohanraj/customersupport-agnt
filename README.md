## Overview

The Customer Support Agent is a sophisticated AI system designed to handle tier-1 customer support for a SaaS product. It efficiently manages tasks such as answering FAQs, triaging support tickets, and escalating complex issues to human agents. The agent operates through multiple communication channels, including web chat and messaging platforms integrated with Zendesk and Slack. At its core, the system utilizes the GPT-4o-mini model for natural language processing, supported by Redis for maintaining conversation context across interactions. The architecture is designed to handle medium load efficiently, with scalability options through horizontal scaling of its core components. Security and compliance are prioritized, with measures in place to handle PII securely and ensure robust authentication with external tools.

## Deployment

This repository includes two GitHub Actions workflows — **Azure Container Apps** (default)
and GCP Cloud Run (available but inactive).

### One-time Azure setup (automated on connect)

When you call `POST /github/connect` with AgentForge's Azure admin credentials configured,
AgentForge automatically configures OIDC and writes AZURE_* secrets to the GitHub
**Environment** named in `AZURE_GITHUB_ENVIRONMENT` (default: `production`).

The workflow (`.github/workflows/deploy-azure.yml`) uses `environment: production` and
deploys automatically on every push to `main`.

### GCP Cloud Run (available, inactive)

A `.github/workflows/deploy-gcp.yml` is also present. To activate it, add three GitHub Secrets:
- `GCP_WORKLOAD_IDENTITY_PROVIDER`, `GCP_SERVICE_ACCOUNT`, `GCP_REGION`
